# Telegram-бот «Азимут Медлайн»

Бот работает как круглосуточный онлайн-консультант клиники: показывает меню услуг, помогает сориентироваться по ценам и форматам помощи, принимает заявки и отвечает на вопросы через DeepSeek API.

## Что умеет

- Приветствует пользователя с логотипом клиники.
- Показывает меню с разделами:
  - Оформить заявку
  - AI-консультант 24/7
  - Услуги
  - Цены
  - Врачи и направления
  - Контакты
  - Оплата
  - Сайт клиники
- В каждом подразделе есть кнопка «Назад».
- Собирает заявку: имя, телефон, направление, комментарий.
- При наличии `ADMIN_CHAT_ID` отправляет заявку администратору в Telegram.
- Отвечает через DeepSeek API, не раскрывая ключ пользователю.

## Важно про токены

Не храните реальные токены в коде и не коммитьте `.env`.

Так как токены уже были отправлены в чат, лучше перевыпустить:

- Telegram bot token в BotFather
- DeepSeek API key в кабинете DeepSeek

После перевыпуска добавьте новые значения только локально или в переменные окружения сервера.

## Локальная настройка

Создайте файл `telegram-bot/.env`:

```bash
cp telegram-bot/.env.example telegram-bot/.env
```

Заполните:

```text
TELEGRAM_BOT_TOKEN=новый_токен_бота
DEEPSEEK_API_KEY=новый_deepseek_api_key
ADMIN_CHAT_ID=
```

Файл `.env` игнорируется git.

## Запуск локально

Node.js нужен версии 18 или выше.

```bash
cd telegram-bot
set -a
source .env
set +a
npm start
```

Бот использует long polling, поэтому процесс должен быть постоянно запущен.

## Проверка синтаксиса

```bash
cd telegram-bot
npm run check
```

## Как получить ADMIN_CHAT_ID

1. Напишите любое сообщение боту.
2. Временно добавьте логирование `message.chat.id` или проверьте updates через Telegram Bot API.
3. Укажите id в `ADMIN_CHAT_ID`.

Если `ADMIN_CHAT_ID` пустой, заявки будут выводиться в консоль процесса.

## Где менять меню

Основные меню находятся в `bot.js` в объекте:

```js
MENU
```

## Где менять AI-поведение

Системный промпт находится в `bot.js`:

```js
SYSTEM_PROMPT
```

## Продакшен-запуск

Для круглосуточной работы нужен сервер, VPS, Render/Railway/Fly.io или другой хостинг, где Node-процесс не засыпает.

Пример через `pm2`:

```bash
npm i -g pm2
cd telegram-bot
set -a
source .env
set +a
pm2 start bot.js --name azimut-telegram-bot
pm2 save
```

Для serverless/Vercel лучше делать webhook-версию, потому что long polling требует постоянно запущенного процесса.
