## Что осталось сделать

1. Заменить фотографии специалистов на реальные фотографии врачей и руководителя, если клиника передает утвержденные фото.
2. Уточнить реальные ФИО, должности, стаж, специализации и тексты карточек специалистов.
3. Добавить/обновить лицензии, сертификаты и юридические материалы после передачи актуальных файлов.
4. Финально подключить amoCRM: нужен admin-доступ или официальный access token, затем тестовая заявка с сайта и проверка сделки в воронке.
5. После замены реальных фото и документов провести финальную проверку страниц, изображений, микроразметки, форм и мобильной версии.

## Изменение ТЗ: amoCRM вместо Bitrix24

В исходном направлении CRM фигурировал Bitrix24, но по требованию заказчика работа была переориентирована на amoCRM. Это не является невыполнением оффера: CRM-часть была изменена по вводным заказчика.

Что уже подготовлено:

- `js/crm-bridge.js` собирает заявки сайта в структуру amoCRM.
- `api/amo-lead.js` является серверным шаблоном для создания сделки через amoCRM API.
- `js/forms.js` отправляет формы через CRM-слой.
- `js/chatbot.js` умеет передавать лиды из AI-чата в CRM-слой.
- Подготовлен документ `docs/amo-crm-funnel-2026-07-17.md` с разбором воронок и полей.

Блокер: нельзя безопасно хранить токен amoCRM в клиентском JavaScript или на GitHub Pages. Для production-запуска нужен серверный endpoint с переменными окружения и официальный токен интеграции.

## SSL: самый дешевый и простой вариант

Сейчас покупать SSL-сертификат не нужно. Домен уже открывается по HTTPS, а HTTP перенаправляется на HTTPS. Для GitHub Pages HTTPS доступен бесплатно, а сертификаты обычно обслуживаются автоматически.

Если позже понадобится CDN, DNS-защита или дополнительные правила безопасности, можно подключить Cloudflare Free. Это тоже 0 ₽, но добавит лишний слой настроек. На текущий момент самый простой вариант: оставить GitHub Pages + включенный HTTPS.

Официальные справки:

- GitHub Pages HTTPS: https://docs.github.com/en/pages/getting-started-with-github-pages/securing-your-github-pages-site-with-https
- Let's Encrypt: https://letsencrypt.org/
- Cloudflare Universal SSL: https://developers.cloudflare.com/ssl/edge-certificates/universal-ssl/
