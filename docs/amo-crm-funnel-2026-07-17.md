## Рекомендуемый порядок настройки

1. Сначала создать поля карточки клиента.
2. Затем создать воронки и этапы.
3. Потом настроить источники лидов с сайта, чатов, звонков, рекламы и партнеров.
4. После этого включить автоматизации, задачи и запреты на неправильное закрытие.
5. В конце проверить тестовыми лидами: форма сайта, чат, звонок, партнер, ручной лид.

## Воронка 0: Неразобранное

Этапы:

1. Обращения из чатов
2. Обращения на сайте
3. Лид Реклама
4. Лид от партнера
5. Пропущенный звонок
6. КВАЛИФИКАЦИЯ
7. Не удалось связаться
8. Из 0 воронки нельзя закрывать в успешно реализованное
9. Закрыто и не реализовано

Логика:

- все входящие лиды сначала попадают сюда;
- успешного закрытия из этой воронки быть не должно;
- после квалификации лид переводится в профильную воронку: Амбулатория, Партнеры, Выезды, Стационар или РЦ;
- если связаться не удалось, ставится задача повторного контакта.

5. Запись подтверждена
6. Пациент не пришел
7. Пациент пришел
8. Рекламация
9. Чек
10. Успешно реализовано
11. Закрыто и не реализовано

Автоматизации:

- при `Выставлен счет на оплату` создать задачу “Проверить оплату”;
- при `Оплата получена` создать задачу “Подтвердить запись”;
- при `Запись подтверждена` отправить пациенту напоминание;
- при `Пациент не пришел` создать задачу “Перезаписать/выяснить причину”;
- при `Рекламация` создать задачу руководителю/администратору.

## AnyDesk или альтернатива

AnyDesk использовать можно, но это дорогой путь по времени и токенам: мне придется читать экран, ждать загрузки интерфейса, аккуратно кликать и постоянно перепроверять состояние.

Более быстрый путь:

1. Дать отдельный доступ в amoCRM или временного пользователя с правами администратора CRM.
2. Подготовить импорт/спецификацию воронок и полей по этому документу.
3. Настраивать через API amoCRM, если доступны long-lived token/интеграция.
4. AnyDesk оставить только для финальной проверки UI, прав пользователей и виджетов.

Если из-за VPN amoCRM не открывается на MacBook, лучшая альтернатива AnyDesk:

- открыть amoCRM на втором ПК;
- экспортировать/передать cookies не нужно и небезопасно;
- вместо этого создать временного пользователя для меня/тебя в amoCRM;
- либо использовать официальный API amoCRM с токеном интеграции;
- после настройки временный доступ удалить.

## Минимальная проверка после настройки

1. Создать лид с сайта: должен попасть в “Неразобранное -> Обращения на сайте”.
2. Создать лид из чата: должен попасть в “Обращения из чатов”.
3. Создать пропущенный звонок: должен попасть в “Пропущенный звонок”.
4. Перевести квалифицированный лид в “Амбулатория”.
5. Проверить запрет успешного закрытия из нулевой воронки.
6. Проверить обязательность причины закрытия для неуспешных исходов.
7. Проверить UTM/source/page_url/form_id для сайта.

## Подготовленная интеграция сайта

На сайте добавлен amoCRM-ready слой:

- `js/crm-bridge.js` — собирает заявки с форм и AI-чата в структуру amoCRM.
- `api/amo-lead.js` — серверный endpoint-шаблон для создания сделки через amoCRM API.
- `js/forms.js` теперь использует `window.AzimutCRM` и оставляет старую совместимость.
- `js/chatbot.js` отправляет лиды AI-чата в `window.AzimutCRM.sendChatLead`.

Текущая логика маршрутизации:

- форма сайта -> `Неразобранное / Обращения на сайте`
- AI-чат -> `Неразобранное / Обращения из чатов`

Чтобы включить реальную отправку:

1. В amoCRM выдать админ-доступ или создать официальную интеграцию.
2. На серверной стороне задать переменные окружения:
   - `AMOCRM_BASE_URL=https://lecmedea.amocrm.ru`
   - `AMOCRM_ACCESS_TOKEN=<токен интеграции>`
   - `AMOCRM_PIPELINE_ID=11108006`
   - `AMOCRM_SITE_STATUS_ID=87274338`
   - `AMOCRM_FIELD_MAP=<JSON с ID полей, если названия полей в amoCRM отличаются>`
3. На сайте указать endpoint:
   - через `window.AZIMUT_CRM_ENDPOINT = "https://.../api/amo-lead"`;
   - или через `<meta name="azimut-crm-endpoint" content="https://.../api/amo-lead">`;
   - или через `data-endpoint` у скрипта `crm-bridge.js`.

Важно: access token нельзя класть в GitHub Pages или клиентский JavaScript. Только серверный endpoint.
