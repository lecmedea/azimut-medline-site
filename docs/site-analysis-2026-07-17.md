## Видео: почему выбран MP4

В `assets/video/` лежали:

| Файл | Размер | Вывод |
|---|---:|---|
| `hero-bg.mp4` | 4.1 MB | Лучший вариант для фонового видео |
| `1368210.mp4` | 4.1 MB | Полный дубликат `hero-bg.mp4`, удален |
| `1368210.gif` | 28 MB | Слишком тяжелый для веб-фона, удален |

MP4 лучше GIF для главной: меньше вес, аппаратное декодирование в браузерах, нормальный `autoplay muted loop playsinline`, можно задать `poster`. GIF держал бы LCP/первую загрузку хуже и быстрее раздувал бы трафик.

## Индексация Google

Проверки:

- `https://azimutclinic.ru/` отдает HTTP 200.
- `https://azimutclinic.ru/sitemap.xml` отдает HTTP 200.
- `robots.txt` разрешает обход сайта и указывает sitemap.
- Поиск `site:azimutclinic.ru` не показывает нормальную выдачу страниц сайта.
- Search Console для `sc-domain:azimutclinic.ru` открыл страницу `not-verified`: у текущего аккаунта нет доступа к этому ресурсу.

Что это значит: я не могу честно нажать “Запросить индексирование” или отправить sitemap без владельческого доступа к Search Console. По документации Google URL Inspection/индексация работает через Search Console для подтвержденных ресурсов, а попадание в Google также зависит от доступности страниц, ссылок, качества и отсутствия запретов.

Источники:

- https://support.google.com/webmasters/answer/9012289?hl=ru
- https://developers.google.com/search/docs/fundamentals/get-on-google?hl=ru

## Что нужно сделать для индексации дальше

1. Подтвердить домен в Google Search Console.
   - Лучший способ: DNS TXT-запись для `azimutclinic.ru`.
   - Альтернатива: URL-prefix `https://azimutclinic.ru/` через HTML-файл или meta-tag, если есть доступ к файлам сайта.

2. После подтверждения:
   - добавить sitemap `https://azimutclinic.ru/sitemap.xml`;
   - запросить индексирование главной через URL Inspection;
   - запросить индексацию ключевых страниц услуг;
   - проверить отчет “Страницы” на исключения;
   - подключить/проверить Google Business Profile для клиники.

3. Усилить внешние сигналы:
   - добавить ссылку на сайт из Яндекс Бизнес, 2ГИС, Zoon/ProDoctorov, VK, Telegram, GitHub Pages canonical;
   - сделать несколько качественных ссылок с партнерских ресурсов;
   - добавить NAP-консистентность: одинаковые название, адрес, телефон.

## Глубокие улучшения сайта

### Высший приоритет

1. Search Console и индексация.
   Пока домен не подтвержден, Google остается черным ящиком. Это блокер SEO.

2. Страницы услуг под поисковые кластеры.
   Сейчас сайт выглядит красиво, но ему нужны посадочные страницы под реальные запросы:
   - консультация психиатра;
   - консультация психотерапевта;
   - семейная терапия;
   - детский психолог/подростковый специалист;
   - тревога, панические атаки, депрессия, бессонница, эмоциональное выгорание;
   - выезд психиатра/психотерапевта, если услуга действительно есть.

3. Медицинское доверие.
   Нужны отдельные блоки:
   - лицензия и реквизиты;
   - врачи с образованием, стажем, сертификатами;
   - цены/диапазоны;
   - как проходит прием;
   - противопоказания/экстренные случаи;
   - правила конфиденциальности и обработки персональных данных.

4. Сквозная аналитика и amoCRM.
   Все CTA должны попадать в amoCRM с UTM, источником, страницей входа, формой и типом услуги.

### Средний приоритет

5. Производительность изображений.
   Баннеры уже облегчены. Следующий шаг - WebP/AVIF версии и `<picture>` для тяжелых hero/assets.

6. Локальное SEO.
   Добавить расширенную `MedicalClinic`/`LocalBusiness` schema с адресом, телефоном, часами работы, координатами и `sameAs`.

7. Контентная стратегия.
   Блог стоит привязать к воронке: статьи должны вести не просто к чтению, а к конкретным услугам и заявкам.

8. Доступность.
   Базовый skip-link добавлен. Дальше стоит пройти полный WCAG-аудит: контраст, фокус, aria-label, формы, ошибки, клавиатура.

### Низкий приоритет

9. Микроанимации и визуальная полировка.
   Уже есть хорошая основа. Новые эффекты стоит добавлять только после проверки Core Web Vitals.

10. Расширение персонализации.
   Например, показывать разные CTA/баннеры для родителя, взрослого пациента, семьи и корпоративного клиента.

## Риски

- Search Console недоступна текущему аккаунту: это главный внешний блокер.
- GitHub-ветка локально сильно расходится с `origin/main`, поэтому публикация должна идти через текущий REST-скрипт проекта или аккуратное выравнивание веток.
- Медицинская тематика требует осторожной формулировки: не обещать “излечение”, не ставить диагнозы, явно отделять информационные материалы от консультации врача.
