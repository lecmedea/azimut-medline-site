- Неразобранное — `11108006`;

Лишние амбулаторные этапы удалены из «Неразобранного», добавлена «Квалификация». Остальные пять воронок заполнены целевыми этапами из Excel. Системные этапы «Неразобранное», «Успешно реализовано» и «Закрыто и не реализовано» amoCRM добавляет автоматически.

Telegram-канал «Азимут Клиник» уже виден в amoCRM как источник мессенджера. Он использует единственный Telegram webhook, поэтому нельзя без проверки назначать второй webhook или включать параллельный polling: это разорвёт существующую связку amoCRM. Код бота оставлен как серверный вариант, а рабочий канал должен настраиваться через Salesbot amoCRM.

## Код синхронизации

- `api/amo-lead.js` — серверный endpoint Vercel, создаёт/находит контакт, создаёт сделку, выбирает этап сайта или чата и добавляет примечание с UTM. Даже если amoCRM временно недоступна, заявка продолжает уходить в реестр и на email.
- `api/amo-webhook.js` — защищённый обратный вебхук для заявок, созданных непосредственно в amoCRM.
- `js/crm-bridge.js` — безопасно передаёт заявки сайта на сервер; access token не попадает в браузер.
- `js/forms.js` — формы теперь используют `AzimutCRM` и показывают подтверждение только после ответа CRM.
- `telegram-bot/bot.js` — заявки из `@azimut_clinic_bot` отправляются в тот же endpoint и попадают в «Обращения из чатов».
- `google-apps-script/Code.gs` — единый реестр, дедупликация, листы по платформам и уведомления на `info@azimutclinic.ru`.

## Что осталось включить в интерфейсах

1. Создать приватную официальную интеграцию amoCRM и получить долгосрочный `AMOCRM_ACCESS_TOKEN`. Если amoCRM покажет отказ от технической поддержки, требуется отдельное подтверждение владельца аккаунта до принятия.
2. Google Web App опубликован, защищённый `LEADS_WEBHOOK_SECRET` записан в Script Properties. URL Web App: `https://script.google.com/macros/s/AKfycbznsQAlR3Iho0-JPwxZq11_7-LhcoFnq2yCPS_8glcxQAnm2G65ZIXwu_gdkApU2AONAg/exec`.
3. В Vercel для Production и Preview заданы `LEADS_WEBHOOK_URL` и `LEADS_WEBHOOK_SECRET`; production пересобран. После создания интеграции остаётся добавить `AMOCRM_ACCESS_TOKEN`, а при включении обратного webhook — `AMO_WEBHOOK_KEY`.
4. В amoCRM добавить URL `/api/amo-webhook?key=...` и настроить Salesbot существующего Telegram-канала. Не менять Telegram webhook напрямую.
5. При необходимости заполнить `AMOCRM_FIELD_MAP` реальными ID пользовательских полей.
6. Отправить две тестовые заявки: с сайта и из Telegram; проверить amoCRM, email и обе строки в Google Sheets.

Секреты нельзя добавлять в GitHub, HTML или клиентский JavaScript.
