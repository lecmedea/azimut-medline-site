# Реализация amoCRM и Telegram — 18 июля 2026

## Настроено в amoCRM

В аккаунте `lecmedea.amocrm.ru` сохранены шесть воронок по таблице:

- Неразобранное — `11108006`;
- Амбулатория — `11115734`;
- Партнеры — `11115738`;
- Выезды — `11115742`;
- Стационар — `11115746`;
- РЦ — `11115750`.

Лишние амбулаторные этапы удалены из «Неразобранного», добавлена «Квалификация». Остальные пять воронок заполнены целевыми этапами из Excel. Системные этапы «Неразобранное», «Успешно реализовано» и «Закрыто и не реализовано» amoCRM добавляет автоматически.

Telegram-канал «Азимут Клиник» уже виден в amoCRM как источник мессенджера. Отдельный Salesbot в разделе «Агенты и боты» пока не создан.

## Код синхронизации

- `api/amo-lead.js` — серверный endpoint Vercel, создаёт/находит контакт, создаёт сделку, выбирает этап сайта или чата и добавляет примечание с UTM.
- `js/crm-bridge.js` — безопасно передаёт заявки сайта на сервер; access token не попадает в браузер.
- `js/forms.js` — формы теперь используют `AzimutCRM` и показывают подтверждение только после ответа CRM.
- `telegram-bot/bot.js` — заявки из `@azimut_clinic_bot` отправляются в тот же endpoint и попадают в «Обращения из чатов».

## Что останется сделать после получения токенов

1. В Vercel задать `AMOCRM_ACCESS_TOKEN` официальной интеграции amoCRM.
2. На хостинге Telegram-бота задать `TELEGRAM_BOT_TOKEN` и `AMO_LEAD_ENDPOINT`.
3. При необходимости заполнить `AMOCRM_FIELD_MAP` реальными ID пользовательских полей.
4. Отправить две тестовые заявки: с сайта и из Telegram; проверить этапы `87274338` и `87274334`.

Секреты нельзя добавлять в GitHub, HTML или клиентский JavaScript.
